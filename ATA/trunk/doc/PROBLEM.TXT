*******************************************************************************
* If the report.* files are missing without COMPELLING reason I'll simply     *
* ignore the whole problem report without notice.			      *
*******************************************************************************

I REQUIRE you to attach the following data to EACH bug or problem report:

Make sure that you run the STANDARD version of DaniS506.ADD rather than the ones
from the /nodebug or /nodebug/cmprssed paths in the distribution archives! The
latter two are unsupported and meant to be used on severely space constrained
boot media only. I've included them for your convenience only and it's up to YOU
to take possible risks doing so.

Add the option /DEBUG:-31 to the DaniS506.ADD line in CONFIG.SYS, and reboot.
Then grab

 - the output from "COPY IBMS506$ report.trc"
 - the output from "DiskInfo cviI > report.txt"
 - the output from "DumpIDE b > report.dmp"

If there are non-zero numbers in the "Total errors" lines of the output from
"DiskInfo cv", run and grab

 - the output from "SmartCtl -a DiskID > report.smart"

where 'DiskID' is the id of the disk with errors, f.e. '0m' if DiskInfo shows
errors for unit '0/m'.

If you are experiencing problems related to any kind of ATAPI unit (CD, DVD,
burners, ...), then grab

 - the output from "COPY ATAPI$ report.atapi"

If you cannot boot from disk using DaniS506.ADD, try the above from a
bootable floppy, CD or ZIP.

If this fails as well, try again with the additional option /FORCEGBM /!BM

If all of the above fails, but a boot with IBM1S506.ADD works, then grab
at least

 - the output from "DumpIDE b > report.dmp"
 - the output from "BldLevel DaniS506.ADD > report.bld"

Send the info to "Daniela Engert" <dani@ngrt.de> please. Don't forget to provide
a valid "reply-to" field in the header of your email, I won't edit it by hand!

If you are experiencing problems with CD burning try disabling busmaster DMA
(/A:x /U:y /!BM) first; some writers have problems with DMA mode.
